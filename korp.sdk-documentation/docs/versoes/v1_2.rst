v1.2
====

| Para atualizar seu serviço para a versão nova, execute o seguinte comando

.. code-block:: bash

    go get bitbucket.org/viasoftkorp/korp.sdk

Breaking changes
^^^^^^^^^^^^^^^^

* Leitura da porta do serviço

| A leitura da porta em que o serviço rodará não virá mais da propriedade ServicePort do Consul. Para especificar a porta, você terá que configurar uma variável de ambiente ``HTTP_PORT``.
| Em modo de desenvolvimento, é recomendado que essa variável não seja preenchida, pois nesse caso o SDK alocará uma porta aleatória **automaticamente**.

Migrando v1.1 para v1.2
^^^^^^^^^^^^^^^^^^^^^^^

* No arquivo main

#. Remover inicialização e configurações manuais do app.fiber. Utilizar o método ``fiber_helper.NewFiberApp()`` que fará todas essas configurações.
#. Remover função ``getCORS()``.

.. warning::

    Ao migrar para o método ``fiber_helper.NewFiberApp()``, verifique se o não está sendo utilizada uma configuração customizada para o fiber.New

    .. code-block:: go

        func main() {

            // configuração customizada, nesse caso deve-se passar a mesma coisa para fiber_helper.NewFiberApp()
            app := app := fiber.New(fiber.Config{
                BodyLimit: 10 * 1024 * 1024
            })

            app := fiber_helper.NewFiberApp(fiber.Config{
		        BodyLimit: 10 * 1024 * 1024,
            })
        }

O seu arquivo main.go precisa ficar com a estrutura igual a https://bitbucket.org/viasoftkorp/viasoft.projecttemplates.golang/src/master/main.go

Adição
^^^^^^^^^

Pacote fiber_helper
------------------------

* Método ``NewFiberApp`` responsável por facilitar a criação e configuraão de um ``*fiber.App`` utilizado nos serviços Korp

.. code-block:: go

    import (
        "bitbucket.org/viasoftkorp/korp.sdk/fiber_helper"
    )

    func main() {
        app := fiber_helper.NewFiberApp()
    }

.. note::
    
    É possível especificar uma configuração customizada para o fiber

    .. code-block:: go

        fiber_helper.NewFiberApp(fiber.Config{
            BodyLimit: 10 * 1024 * 1024,
        })


* Método ``Listen`` responsável para iniciar o listener do fiber

.. code-block:: go

    import (
        "bitbucket.org/viasoftkorp/korp.sdk/fiber_helper"
    )

    func main() {
        app := fiber_helper.NewFiberApp()
        fiber_helper.Listen(app)
    }

Melhorias
^^^^^^^^^

Pacote environment
------------------

| Nova função ``GetServicePort`` para busca de porta que o serviço vai rodar.

.. code-block:: go

    import (
        "bitbucket.org/viasoftkorp/korp.sdk/environment"
    )

    func main() {
        port, err := environment.GetServicePort()        
    }

.. warning::

    Se você está utilizando esse método manualmente para iniciar um router (gin, fiber, etc), certifique-se que o formato utilizado na função ``fmt.Sprintf`` é um ``%d`` e *não* ``%s``. Exemplo:

    .. code-block:: go

        func main() {
            port, err := environment.GetServicePort()
            if err != nil {
                log.Fatalf("Couldn't get servicePort due to error: %s", err)
            }

            //INCORRETO
            //servicePort := fmt.Sprintf(":%s", port)

            //CORRETO
            servicePort := fmt.Sprintf(":%d", port)
            err = router.Run(servicePort)
            if err != nil {
                log.Fatalf("error while initiating router, %s", err)
            }
        }

Suporte a certificado auto-assinado para OnPremise
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

* Clonar o repositório `viasoft.projecttemplates <https://bitbucket.org/viasoftkorp/viasoft.projecttemplates.golang>`_.

    * Copiar o arquivo ``.gitattributes`` para o seu repositório e comitar. Se esse arquivo já existir no seu projeto, deve substituir.
    * Copiar o arquivo ``entrypoint.sh``, trocar ``<ServiceName>`` pelo nome do seu serviço e comitar.
    * Copiar o arquivo ``Dockerfile``, trocar ``<ServiceName>`` pelo nome do seu serviço e comitar.
